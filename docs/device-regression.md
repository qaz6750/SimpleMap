# Android 真机回归

## 版本兼容矩阵

生产构建使用 `minSdk 26`、`compileSdk 36`、`targetSdk 36`。Android 17（API 37）截至 2026 年 7 月仍为 Beta，仅用于前向兼容测试；在其正式稳定且高德导航 SDK 明确支持前，不把生产构建迁移到 API 37 或 AGP 9。

| 系统版本 | 必测兼容点 |
| :--- | :--- |
| Android 8.0–8.1（API 26–27） | 最低版本安装与冷启动、通知渠道、位置权限、地图与导航 View 生命周期。 |
| Android 9（API 28） | 刘海屏竖横屏切换、TLS 网络请求、前后台地图恢复。 |
| Android 10–11（API 29–30） | `location` 类型前台服务、仅使用期间和单次位置授权、手势与三键导航模式。 |
| Android 12–12L（API 31–32） | 不可变 `PendingIntent`、后台前台服务启动限制、启动画面、分屏与折叠/大屏窗口。 |
| Android 13（API 33） | `POST_NOTIFICATIONS` 允许与拒绝两条路径；拒绝后导航仍运行且可从系统前台服务管理器停止。 |
| Android 14（API 34） | `FOREGROUND_SERVICE_LOCATION`、仅在可见界面且已有位置权限时启动导航服务、精确/模糊位置切换。 |
| Android 15（API 35） | 强制 edge-to-edge、状态栏隐藏、手势小白条与三键导航对比度、16 KB 页面设备启动。 |
| Android 16（API 36） | 预测性返回、`sw600dp` 以上窗口忽略方向/宽高比限制、自由窗口缩放、全面屏与大字体。 |
| Android 17 Beta（API 37） | 大屏方向限制不再允许退出；检查本地网络权限、证书透明度、原生动态加载只读限制和应用内存限制。 |

当前高德聚合 SDK 仅打包 `arm64-v8a`。Debug APK 内原生库已通过 Build Tools 36 的 `zipalign -c -P 16` 检查，ELF 加载段至少按 16 KB 对齐；每次升级高德 SDK 后都要重新执行该检查。

## 准备

1. 在 `local.properties` 配置有效的 `AMAP_API_KEY`，不要提交该文件。
2. 连接一台已授权的 Android 8.0 或更高版本真机，并启用 GPS、网络和 USB 调试。
3. 运行 `scripts/device-regression.sh all`。自动化测试结束后，应用会以全新数据启动。
4. 在设备上阅读并同意隐私说明。确认同意前没有地图、定位、搜索或导航 SDK 请求。

## 高德能力

- 地图：首帧正常、缩放和拖动流畅，前后台切换后不黑屏。
- 定位：仅授予精确位置后显示当前位置；关闭系统定位时显示明确提示。
- 搜索：地点与公交搜索返回当前城市结果，地点详情、距离和电话合理。
- 路线：驾车、步行和骑行可规划；驾车偏好和途经点生效。
- 导航：GPS 和模拟导航均能开始，转向图标、剩余距离、ETA、路况和语音更新。
- 弱 GPS：进入室内或遮挡环境后显示弱信号状态，恢复后自动消失。
- 离线地图：城市列表可加载，下载、暂停、删除及仅 Wi-Fi 设置生效。

## 主题、路线预设与语音

- 将主题设为“跟随系统”，切换系统深浅色后确认应用界面、首页底图和导航底图同步更新。
- 将主题设为“按时间自动”，分别在 05:59、06:00、18:59 和 19:00 检查日夜边界；进入名称包含“隧道”或“地道”的路段时导航底图临时切为夜间，驶出后恢复。
- 卫星底图开启时保持卫星图；关闭卫星图后恢复当前自动日夜底图。
- 依次选择“通勤”“高速优先”“新能源省电”，确认分别应用躲避拥堵、高速优先，以及躲避拥堵/不走高速/少收费组合，重启后偏好保持。
- 将语音级别切为“详细播报”“简洁播报”“静音”，确认高德内置导航语音对应变化；“静音”下不播报任何导航或自定义提示。
- 启用跨午夜静音时段（例如 22:00 至 07:00），确认常规转向播报暂停；开启“重要提示语音”时严重拥堵仍播报，关闭后不再播报。
- 开始后台导航后锁屏并跨过静音时段或日夜边界，确认语音与导航底图状态在重新打开应用后正确。

## 车机与响应式 UI

- 在 `640×320`、`640×360` 和宽屏车机上，导航左侧信息卡不遮挡车辆与前方路线。
- 路口放大图完整显示上下边缘；出现时车速、限速、GPS 和当前道路仍可见。
- 接近多车道路口时显示高德 SDK 车道信息，推荐车道以蓝色高亮；驶离路口后车道条及时消失。
- 道路封闭优先于严重拥堵，严重拥堵优先于普通路线提示；同一时刻只显示一个安全提示。
- 未拖动地图时可单击打开导航设置；拖动后显示跟随、设置和结束，触控区域不小于 48dp。
- 路况柱完全位于屏幕内，不与左侧信息栏、车速、GPS 或底部道路名重叠。
- 横屏路线规划的输入、偏好和方案位于左侧信息栏，地图右侧仍能看到完整起终点和路线。
- 在 Android 16 的 `sw600dp` 以上设备旋转屏幕并调整自由窗口尺寸，确认系统忽略方向限制时界面仍能正确重排。
- 在 `320×480`、`360×640`、`640×320` 下分别使用默认、1.3 倍和 2.0 倍字体检查文字不裁切、控件不重叠。
- 打开搜索软键盘后，全局底栏隐藏；取消按钮、输入框和最后一条可滚动结果仍可点击。
- 系统返回键依次关闭搜索、地点详情和路线规划，不直接退出地图；公交线路结果可展开查看站点。

## 后台导航

1. 开始一次真实 GPS 导航，确认常驻通知显示目的地和“结束导航”。
2. 返回桌面并从最近任务划掉 SimpleMap，保持屏幕关闭至少两分钟。
3. 确认语音导航和通知继续，系统定位指示仍显示。
4. 点击通知重新打开应用，确认当前道路、剩余距离和 ETA 延续，没有重新规划起点。
5. 再次划掉应用，点击通知的“结束导航”，确认通知、定位和语音均停止。
6. 重新打开应用，在“行程”中确认后台完成或手动结束的行程只记录一次。

## 常用地点

- 旧收藏首次打开后位于“收藏夹”。
- 将地点设置为“家”或“公司”后，分组立即更新；每组只保留一个地点。
- 点击地点主体可直接进入路线规划，重启应用后分组保持不变。
- 清除本地数据后，家、公司和收藏夹全部清空。

## 日志

问题复现后运行 `scripts/device-regression.sh logs`。日志写入 `build/device-regression-log.txt`，提交或分享前检查其中不包含位置、搜索词或其他用户数据。
